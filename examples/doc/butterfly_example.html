  • ¶

    Example data sets for function-level control flow graph.

  • ¶

    Function records contain an id, their name, and the name of the module they belong to.

    interface FunctionRecord {
    id: number;
    name: string;
    module: string; // Not used in this example
}
  • ¶

    Function call links records contain the function id for the source and target of the call. source is the caller and target is the callee. value is the total amount of time for the function call (and any nested calls).

    interface CFGLinkRecord {
    source: number;
    target: number;
    value: number;
}
